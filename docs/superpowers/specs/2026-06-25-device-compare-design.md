# หน้าเทียบเครื่องวัด (Device Compare) — Design

วันที่: 2026-06-25
สถานะ: approved (brainstorm), กำลัง implement

## ปัญหา / เป้าหมาย

ห้องเก็บสีย้อมตอนนี้มีเซนเซอร์หลายเครื่องในห้องเดียวกัน เครื่องที่อยู่ห้องเดียวกัน
ควรอ่านค่าใกล้เคียงกัน ผู้ใช้อยากมีหน้าที่:

1. **เช็คว่าเครื่องอ่านตรงกันไหม** — ถ้าเครื่องใดอ่านต่างจากตัวอื่นมาก = อาจเซนเซอร์เพี้ยน
   หรือต้องสอบเทียบ (จุดประสงค์หลักที่ผู้ใช้ระบุ)
2. **ดูค่าต่างสูงสุด + เครื่องที่หลุดกลุ่ม** — มีตัวเลขบอกว่าต่างกันกี่หน่วย เครื่องไหนผิดปกติ

(ผู้ใช้เลือก "ทั้งสองอย่าง": กราฟซ้อนเทียบ + ตัวเลขค่าต่าง/เครื่องที่หลุด)

## ขอบเขต (scope)

- หน้าใหม่ `compare.html` แยกจาก dashboard เดิม (ไม่แตะ logic เดิมของ index/app.js
  นอกจากเพิ่มลิงก์นำทาง)
- สลับดูได้ 3 ค่า: **ความชื้น (hum) / อุณหภูมิ (temp) / จุดน้ำค้าง (dew)**
- ใช้แถบช่วงเวลาเดิม: วัน / สัปดาห์ / เดือน / ปี + ปุ่มเลื่อนช่วง
- เทียบเฉพาะเครื่องที่มีข้อมูลในช่วงที่เลือก ถ้าเหลือเครื่องเดียว → แจ้งว่าเทียบไม่ได้

ไม่อยู่ในขอบเขต: export, แจ้งเตือนอัตโนมัติ, เลือก subset เครื่องเอง (เทียบทุกเครื่องที่มีข้อมูล)

## สถาปัตยกรรม

Netlify static (`public/`) + serverless function (`netlify/functions/`) + Neon Postgres
ตารางที่ใช้: `readings(device_mac, ts, temp, hum, dew, vpd)`, `devices(device_mac, name)`

ส่วนประกอบใหม่:

| ไฟล์ | หน้าที่ |
|---|---|
| `shared/agreement.js` | ฟังก์ชัน pure `analyzeAgreement()` — คำนวณค่าต่าง/เครื่องหลุด/ระดับ (เทสต์ได้) |
| `netlify/functions/compare.js` | endpoint `/api/compare` — query + เรียก analyzeAgreement |
| `public/compare.html` | โครงหน้า |
| `public/compare.js` | logic หน้า: fetch, สลับค่า, กราฟซ้อน, ตาราง |
| `public/index.html` | เพิ่มปุ่ม "⚖️ เทียบเครื่อง" ที่ topbar |

## Endpoint `/api/compare`

`GET /api/compare?range=day|week|month|year&date=YYYY-MM-DD`

ใช้ `resolvePeriod(range, date)` เดิมหา {start, end, grain} แล้ว group ต่อเครื่องเป็น
"ช่อง" (bucket) เวลาเดียวกันให้ทุกเครื่อง:

| ช่วง (grain) | bucket |
|---|---|
| วัน (raw) | รายชั่วโมง `date_trunc('hour', ts)` (≤24 ช่อง) |
| สัปดาห์/เดือน (day) | รายวัน `date_trunc('day', ts)` |
| ปี (month) | รายเดือน `date_trunc('month', ts)` |

### Query (2 ครั้ง)

1. **series** — group by (device_mac, bucket): คืน avg ของ hum/temp/dew ต่อเครื่องต่อช่อง
   เฉพาะช่วง [start, end)
2. **summary** — group by device_mac: count + avg/min/max ของ hum/temp/dew ทั้งช่วง
   join `devices` เพื่อเอาชื่อ

### Response

```json
{
  "range": "month", "date": "2026-06-01",
  "start": "...", "end": "...", "grain": "day", "label": "2026-06",
  "buckets": ["2026-06-16", "2026-06-17", ...],   // union ของ bucket key เรียงเวลา
  "devices": [
    { "mac": "...", "name": "...", "n": 1234,
      "hum": {"avg":58.1,"min":51,"max":63},
      "temp": {"avg":29.2,"min":26,"max":32},
      "dew":  {"avg":19.8,"min":17,"max":22} }
  ],
  "series": [ {"mac":"...", "t":"2026-06-16", "hum":58.0, "temp":29.1, "dew":19.7}, ... ],
  "agreement": {
    "hum":  {"gapAvg":1.8,"gapMax":4.0,"outlier":"<mac|null>","level":"good"},
    "temp": {...},
    "dew":  {...}
  }
}
```

`series` เป็น flat array — ฝั่งหน้า pivot ตาม `t` เป็นเส้นกราฟต่อเครื่องเอง
`agreement` คำนวณครบทั้ง 3 ค่าไว้แล้ว → สลับค่าได้ทันทีโดยไม่ต้อง fetch ใหม่

## ตรรกะ `analyzeAgreement` (pure, ใน shared/)

อินพุต: รายการ series ที่ align ตาม bucket แล้ว (ต่อ bucket มีค่าของแต่ละเครื่อง),
สรุปต่อเครื่อง, ชื่อ metric, เกณฑ์

ขั้นตอน:
1. แต่ละ bucket ที่มี **≥2 เครื่อง** มีค่า → `gap = max(ค่าเครื่อง) − min(ค่าเครื่อง)`
2. `gapAvg` = เฉลี่ยของ gap ทุก bucket, `gapMax` = มากสุด
3. **outlier** = เครื่องที่ค่าเฉลี่ยทั้งช่วงห่างจาก median ของเครื่องอื่นมากสุด
   (และเกินครึ่งหนึ่งของเกณฑ์ "ควรตรวจ") — ไม่งั้น null
4. `level` จาก `gapMax` เทียบเกณฑ์: good / warn / bad

ถ้ามีเครื่องเดียว (หรือไม่มี bucket ที่ ≥2 เครื่อง) → `{gapAvg:null, gapMax:null, outlier:null, level:"single"}`

### เกณฑ์เริ่มต้น (อิงความแม่นเซนเซอร์ Inkbird IBS-TH3: ±3%RH, ±0.5°C)

| ค่า | good 🟢 | warn 🟡 | bad 🔴 |
|---|---|---|---|
| hum | ≤ 3% | 3–6% | > 6% |
| temp | ≤ 0.5° | 0.5–1.5° | > 1.5° |
| dew | ≤ 0.5° | 0.5–1.5° | > 1.5° |

## โครงหน้า (compare.html / compare.js)

1. **topbar** — brand + theme toggle + ลิงก์ "← แดชบอร์ด"
2. **metric switcher** — pills: ความชื้น / อุณหภูมิ / จุดน้ำค้าง (เปลี่ยน = re-render จาก data เดิม)
3. **range bar + period nav** — เหมือน dashboard (วัน/สัปดาห์/เดือน/ปี + ‹ ปุ่มเลือก ›)
4. **verdict card** — สรุปตามค่าที่เลือก:
   - 🟢 "เครื่องวัดตรงกันดี" / 🟡 "ต่างกันเล็กน้อย" / 🔴 "⚠️ มีเครื่องอ่านต่างผิดปกติ — {ชื่อเครื่อง}"
   - บรรทัดรอง: "ต่างกันเฉลี่ย X · มากสุด Y"
   - กรณีเครื่องเดียว: "ช่วงนี้มีข้อมูลเครื่องเดียว เทียบไม่ได้"
5. **overlay line chart** — 1 เส้น/เครื่อง ตาม bucket, สีต่างกันต่อเครื่อง (palette), แกน y ตามค่าที่เลือก
6. **comparison table** — แถว = เครื่อง; คอลัมน์ = เฉลี่ย / ต่ำ–สูง / Δ จากค่ากลาง; ไฮไลต์เครื่องที่หลุด

## ส่วนขยาย (2026-06-25) — ให้ละเอียดเท่าโปรแกรมหลัก

ผู้ใช้ขอให้หน้า compare ละเอียด/ครบเท่า dashboard หลัก เพิ่ม:

1. **กราฟ resolution เท่าหน้าหลัก** — วัน (grain raw) เปลี่ยนจาก bucket รายชั่วโมง → **ราย 10 นาที**
   ด้วย `date_bin('10 minutes', ts, '2000-01-01')`
   - หมายเหตุข้อมูลจริง: เครื่อง 1 log ทุก 10 นาที แต่เครื่อง 2–3 log **รายชั่วโมง**
     → ตั้ง `spanGaps:true` ในกราฟ เพื่อให้เส้นเครื่องที่ log ห่างยังต่อเนื่อง
   - แกน Y auto-scale (ไม่ fix 0–100) เพื่อให้เห็นความต่างเล็กๆ ชัด
2. **ตารางเทียบรายช่วงเวลา** (เพิ่มจากตารางสรุปต่อเครื่อง = มี "ทั้งสองแบบ")
   - แต่ละแถว = ช่วงเวลา (วัน raw → รวมเป็นรายชั่วโมง, สัปดาห์/เดือน → รายวัน, ปี → รายเดือน)
   - คอลัมน์: ช่วงเวลา · **ต่าง** (pill สี เขียว/เหลือง/แดงตามเกณฑ์) · #1..#N (หัวสั้น เชื่อมสีกับ legend กราฟ)
   - หัวคอลัมน์ใช้ #N + จุดสี เพราะชื่อเครื่องภาษาไทยยาวทำให้ตารางล้นจอมือถือ
3. **Heatmap** (เฉพาะ สัปดาห์/เดือน → วัน×ชั่วโมง, ปี → เดือน×ชั่วโมง — วันไม่มี เหมือนหน้าหลัก)
   - **รายเครื่อง**: heatmap ต่อเครื่อง ใช้ color scale ร่วมกัน (min/max รวมทุกเครื่อง) → เทียบด้วยตาได้
   - **แผ่นความต่าง**: heatmap ที่แต่ละช่อง = ค่าต่าง (max−min ของเครื่องที่มีข้อมูลในช่องนั้น)
     สเกล เทา(ตรงกัน)→เหลือง→แดง(ต่างมาก), normalize ด้วย max(maxGap, เกณฑ์ bad)

Endpoint `/api/compare` เพิ่ม field `heat: [{mac,d,h,hum,temp,dew}]` (ว่างเมื่อ range=วัน)
ตารางช่วงเวลา + heatmap ทั้งหมดคำนวณฝั่ง client จาก data ที่ cache ไว้ → สลับ metric ได้ทันที
